### 配置分区表（可选）

只有 3 种情况才需要自己配置分区表

1. 程序太大，默认 1MB/2MB 装不下
2. 要存大文件（网页、图片、音频）
3. 要用 OTA 升级

一般不用配置，如果要配置 需要找到芯片对应的flash大小

![image-20260401215512571](C:\Users\13981\AppData\Roaming\Typora\typora-user-images\image-20260401215512571.png)

![image-20260401215559362](C:\Users\13981\AppData\Roaming\Typora\typora-user-images\image-20260401215559362.png)

![image-20260401215900406](C:\Users\13981\AppData\Roaming\Typora\typora-user-images\image-20260401215900406.png)

| 选项                                 | 含义                         | 适用场景                                                     |
| ------------------------------------ | ---------------------------- | ------------------------------------------------------------ |
| `Single factory app, no OTA`         | 单工厂应用分区，无 OTA       | 最基础的默认配置，适合普通项目，无需 OTA 升级                |
| `Single factory app (large), no OTA` | 单工厂应用分区（大），无 OTA | 程序需要更大空间的非 OTA 项目                                |
| `Factory app, two OTA definitions`   | 工厂分区 + 两个 OTA 分区     | 支持 OTA 升级，需要 A/B 分区                                 |
| `Two large size OTA partitions`      | 两个大尺寸 OTA 分区          | 双 OTA 分区，固件空间更大                                    |
| `Custom partition table CSV`         | 使用自定义 CSV 文件          | 就是你之前自己编辑的`partitions-4MIB.csv`/`partitions-16MIB.csv` |

**OTA = Over-the-Air（空中下载技术）**

对 ESP32 来说，就是：

**通过 WiFi / 网络，远程给芯片更新程序，不用拔插 USB！**

###### 插入组件

```cmake
#  组件源文件目录
set(src_dirs 
    LED
) 

# 组件头文件目录
set(include_dirs 
    LED
)

# 组件依赖项
set(requires 
    driver
)

# 注册组件
idf_component_register(
    SRC_DIRS ${src_dirs}
    INCLUDE_DIRS ${include_dirs}
    REQUIRES ${requires}
)

# 编译选项：启用快速数学运算、优化级别3、禁用格式化错误和警告
component_compile_options(
    -ffast-math
    -O3
    -Wno-error=format
    -Wno-format
)
```
